# B-001: Dungeon Attacks and Magic Skip Post-Command Monster Turns

## Summary

Player attacks and magic resolved in the portable core without running the original dungeon monster-turn path.

## Reference

- BASIC lines: `1090-1096`, `1668-1669`, `1685-1697`
- Fixed code: `Core/src/ak_game.c` `apply_attack`, `apply_magic`
- Related feature: `docs/features/F-008.md`
- Related remaining food bug: `docs/bugs/B-002.md`

## Expected

In the BASIC listing, combat and magic paths end by jumping to line `1090`. In dungeon mode that path runs monster turns through `GOSUB 4000`.

## Actual

Fixed. `apply_attack` and `apply_magic` now run dungeon monster turns after successful player actions when the player remains in dungeon mode. Fractional dungeon food consumption is still tracked separately in B-002.

## Validation

- Added harness tests proving dungeon attack and magic commands run a monster turn when a monster is active.
- `make -C harness test` passed.
- `swift test` passed.
