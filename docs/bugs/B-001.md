# B-001: Dungeon Attacks and Magic Skip Post-Command Food and Monster Turns

## Summary

Player attacks and magic resolve in the portable core without running the original dungeon post-command path.

## Reference

- BASIC lines: `1090-1096`, `1668-1669`, `1685-1697`
- Current code: `Core/src/ak_game.c` `apply_attack`, `apply_magic`
- Related feature: `docs/features/F-008.md`

## Expected

In the BASIC listing, combat and magic paths end by jumping to line `1090`. In dungeon mode that path applies dungeon food consumption, refreshes status, and runs monster turns through `GOSUB 4000`.

## Actual

`apply_attack` and `apply_magic` return directly after resolving the player action. Monsters do not retaliate or move after those actions, and the dungeon food path is skipped.

## Validation

- Add harness tests proving a dungeon attack runs a monster turn when a monster is active.
- Add harness tests proving a dungeon magic command runs the same post-command turn path.
- Run `make -C harness test` and `swift test`.
