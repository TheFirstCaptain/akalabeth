# B-004: Active Quest Reminders Incorrectly Increase Every Stat

Status: Fixed

## Summary

Pressing space on an active Lord British quest reminder increases all stats, but the original only does this when assigning or advancing a quest.

## Reference

- BASIC lines: `7050-7070`, `7500-7530`
- Current code: `Core/src/ak_game.c` `apply_quest_acknowledgement`
- Current test to update: `Tests/AkalabethMacTests/GameSessionTests.swift` `spaceAcknowledgesQuestPrompt`
- Related feature: `docs/features/F-008.md`

## Expected

For a positive active `TASK`, line `7500` reminds the player what to kill and returns to the command loop. No stat increase occurs. Stat increases happen through line `7070` after first quest assignment and after completed quest escalation.

## Actual

`apply_quest_acknowledgement` increments every stat unconditionally, including when `quest_target > 0`.

## Validation

- Update the existing Swift regression test so active quest acknowledgement preserves stats.
- Add harness tests for first quest assignment, active quest reminder, completed quest escalation, and final victory.
- Run `make -C harness test` and `swift test`.

## Resolution

`apply_quest_acknowledgement` now awards the all-stat increase only when assigning the first quest or advancing a completed quest. Active positive quest reminders leave stats unchanged.
