# B-002: Dungeon Turns Do Not Consume Fractional Food

## Summary

Dungeon movement, turning, pass, combat, and magic do not consume food at the original dungeon rate.

## Reference

- BASIC lines: `1090-1096`
- Current code: `Core/src/ak_game.c` `consume_overworld_food`, dungeon movement/pass paths
- Related feature: `docs/features/F-006.md`

## Expected

The original line `1090` applies `PW(0) = PW(0) - 1 + SGN(INOUT) * .9`. In the overworld this consumes `1` food. In dungeon mode, where `INOUT > 0`, it consumes `0.1` food per command after truncation behavior is applied by line `1092`.

## Actual

The core models food as integer inventory and only calls `consume_overworld_food` for overworld movement, overworld pass, and overworld entry. Dungeon turns do not reduce food.
