# B-002: Dungeon Turns Do Not Consume Fractional Food

## Summary

Dungeon movement, turning, pass, combat, and magic did not consume food at the original dungeon rate.

## Reference

- BASIC lines: `1090-1096`
- Fixed code: `Core/src/ak_game.c` `food_tenths`, dungeon post-command path
- Fixed render code: `Core/src/ak_render.c` main status food formatting
- Fixed persistence code: `Sources/AkalabethMac/GameSession.swift` save/restore state
- Related feature: `docs/features/F-006.md`

## Expected

The original line `1090` applies `PW(0) = PW(0) - 1 + SGN(INOUT) * .9`. In the overworld this consumes `1` food. In dungeon mode, where `INOUT > 0`, it consumes `0.1` food per command after truncation behavior is applied by line `1092`.

## Actual

Fixed. The core now stores canonical food in tenths, keeps `inventory[FOOD]` synchronized as the whole-food compatibility/display value, and consumes `0.1` food for dungeon post-command turns. Main status rendering can show fractional values such as `FOOD=9.9`, and save/restore preserves fractional food.

## Validation

- Added harness tests for dungeon forward, turn, pass, attack, and magic food consumption.
- Added harness starvation coverage for dungeon turns.
- Added render coverage for fractional `FOOD=9.9`.
- Added Swift save/restore coverage for fractional food.
- `make -C harness test` passed.
- `swift test` passed.
