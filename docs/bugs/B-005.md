# B-005: Thrown Axes Are Resolved as Bow Attacks

## Summary

The Swift session adapter consumes an axe for a thrown axe attack, but then sends the core a bow attack.

## Reference

- BASIC lines: `1652`, `1670-1673`
- Current code: `Sources/AkalabethMac/GameSession.swift` `handleAxeStyle`
- Related feature: `docs/features/F-008.md`

## Expected

In the original, choosing axe sets `DAM = 5`. If the player chooses throw, one axe is consumed and the same axe damage is used with ranged targeting up to five squares.

## Actual

`handleAxeStyle` decrements axe inventory and calls `attack(with: AK_GAME_ITEM_BOW)`. This can incorrectly require bow ownership and uses bow damage/rules instead of axe throw rules.

## Validation

- Represent axe throw distinctly in the core command model or command value.
- Add tests proving thrown axe consumes one axe, does not require a bow, uses axe damage, and can target range.
- Run `make -C harness test` and `swift test`.
