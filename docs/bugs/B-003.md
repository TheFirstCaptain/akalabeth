# B-003: Exiting Town Regenerates the Overworld

## Summary

Leaving town rebuilt the world map and relocated the player instead of returning to the existing overworld state.

## Reference

- BASIC lines: `30-50`, `1500`, `60080-60250`
- Fixed code: `Core/src/ak_game.c` `return_to_overworld`, `AK_GAME_COMMAND_EXIT`
- Related feature: `docs/features/F-006.md`

## Expected

The BASIC listing generates the overworld during startup and then treats town as an overlay entered from the current overworld tile. Quitting the shop returns to the command loop and existing overworld map/position.

## Actual

Fixed. `AK_GAME_COMMAND_EXIT` now generates the overworld only for the initial startup town state. If the player entered town from an existing overworld tile, exit preserves the map and player coordinates.

## Validation

- Added harness coverage that captures overworld coordinates and tile data before entering town, exits town, and verifies they are preserved.
- Kept initial town exit generation for startup/new-game flow.
- `make -C harness test` passed.
- `swift test` passed.
