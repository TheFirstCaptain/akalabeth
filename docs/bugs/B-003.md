# B-003: Exiting Town Regenerates the Overworld

## Summary

Leaving town currently rebuilds the world map and relocates the player instead of returning to the existing overworld state.

## Reference

- BASIC lines: `30-50`, `1500`, `60080-60250`
- Current code: `Core/src/ak_game.c` `enter_overworld`, `AK_GAME_COMMAND_EXIT`
- Related feature: `docs/features/F-006.md`

## Expected

The BASIC listing generates the overworld during startup and then treats town as an overlay entered from the current overworld tile. Quitting the shop returns to the command loop and existing overworld map/position.

## Actual

`AK_GAME_COMMAND_EXIT` calls `enter_overworld`, and `enter_overworld` calls `generate_overworld`. This resets the map, castle/town/dungeon placement, and player overworld coordinates.

## Validation

- Add harness coverage that captures overworld coordinates and tile data before entering town, exits town, and verifies they are preserved.
- Keep initial world generation tied to startup/new-game flow rather than shop exit.
- Run `make -C harness test` and `swift test`.
