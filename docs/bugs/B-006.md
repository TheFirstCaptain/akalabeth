# B-006: Bare-Hands Attack Is Not Reachable From the App Prompt

Status: Fixed

## Summary

The original allows an unrecognized attack weapon response to proceed as a bare-hands attack. The app rejects it and keeps prompting.

## Reference

- BASIC lines: `1650-1661`
- Current code: `Sources/AkalabethMac/GameSession.swift` `handleAttackWeapon`
- Related feature: `docs/features/F-008.md`

## Expected

If the player enters an attack weapon choice that does not set a weapon damage value and is not magic, BASIC prints `HANDS` and attacks the adjacent square with base damage `0` plus strength contribution.

## Actual

The Swift prompt returns `AK_GAME_RESULT_INVALID_COMMAND` and sets `statusLine = "WHICH WEAPON"` for unrecognized choices, so bare-hands attack cannot be selected through the app UI.

## Validation

- Add a Swift session test that an unrecognized weapon prompt choice maps to a bare-hands attack.
- Add harness coverage for bare-hands hit/miss behavior in the core.
- Run `make -C harness test` and `swift test`.

## Resolution

Unrecognized attack prompt choices now send a bare-hands attack to the core and show `HANDS`. Harness coverage verifies bare-hands miss and hit behavior using strength-only damage.
