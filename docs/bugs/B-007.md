# B-007: Dungeon Kill Hit-Point Reward Is Not Awarded on Dungeon Exit

## Summary

The original accumulates hit-point rewards for dungeon kills and grants them when leaving the dungeon. The current state model has no equivalent reward accumulator.

## Reference

- BASIC lines: `1665-1667`, `1581-1587`
- Current code: `Core/src/ak_game.c` `kill_monster`, dungeon ladder exit path
- Related feature: `docs/features/F-008.md`

## Expected

When a monster dies, BASIC adds `INT(MN * IN / 2)` to `LK`. When the player exits dungeon level 1, line `1586` adds `LK` to hit points and resets `LK`.

## Actual

`kill_monster` awards gold and quest credit but does not accumulate an HP reward. The level-1 ladder exit path returns to overworld without awarding any accumulated kill reward.

## Validation

- Add an `AkGameState` field or equivalent persisted model for the pending dungeon HP reward.
- Add harness coverage for kill accumulation and level-1 dungeon exit reward.
- Ensure save/restore preserves the pending reward if it can exist across sessions.
- Run `make -C harness test` and `swift test`.
