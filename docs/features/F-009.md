# F-009: Render Command Contract for Apple II-Style Output

## Summary

Define a render-neutral command stream that lets the portable core request Apple II-style text, line art, status panels, monsters, and prompts without depending on macOS UI APIs.

## Reference

- BASIC lines: `100-190`, `200-490`, `3087-3089`, `6000-6060`, `7000-7990`, `60080-60250`
- Screenshots: `OUTSIDE.GIF`, `TOWN.GIF`, `CHEST.GIF`, `HID-TRAP.GIF`, `LB-1.GIF`, `LB-2.GIF`, `LB-3.GIF`, `MNSTR-01.GIF` through `MNSTR-10.GIF`
- Docs: `BOUNDARIES.md`, `docs/modernization/validation-strategy.md`

## Scope

- Add a portable render command model for text cells, inverse text, high-resolution line commands, colors, clears, prompts, and screen modes.
- Map extracted game states to render commands without choosing a final Mac renderer.
- Add reference fixtures for key screens.

## Acceptance Criteria

- Core render APIs expose only portable C types.
- Harness tests can compare representative render command sequences for startup, town, overworld, dungeon, and quest screens.
- The command stream carries enough information for a later Swift/AppKit or Metal-backed renderer to draw the game.
- Screenshot comparisons are documented as manual validation until image-based tests exist.
