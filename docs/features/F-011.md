# F-011: Mac Persistence, Settings, and Distribution Package

## Summary

Add Mac-specific persistence, user settings, app metadata, icons, signing-ready packaging, and release validation for distributing the Apple silicon build.

## Reference

- BASIC lines: no direct save-game source; `README.md` notes no historical save system
- Screenshots: startup and gameplay screenshots for app icon and store-preview reference only
- Docs: `docs/modernization/legacy-inventory.md`, `BOUNDARIES.md`, `DECISION_LOG.md`

## Scope

- Add settings for compatibility preferences such as scale, color treatment, audio toggles, or deterministic debug seeds where appropriate.
- Add save/resume only as an intentional modern feature, separated from historical behavior.
- Add bundle metadata, app icon source, release build instructions, and packaging checks.
- Keep persistence adapters outside `Core/` platform-neutral rule code.

## Acceptance Criteria

- The app stores settings and any save data in an appropriate macOS user location.
- Save/resume behavior, if implemented, is documented as a modern addition rather than original Applesoft behavior.
- Release build instructions produce an Apple silicon app bundle.
- Packaging validation covers clean install, reset/new game, settings persistence, and app relaunch.

## Implementation Status

Implemented.

- `Sources/AkalabethMac/AkalabethPersistence.swift` stores display settings in `UserDefaults` and versioned JSON save documents in `~/Library/Application Support/Akalabeth/SaveGame.json` by default.
- `GameSession.savedSession()` and `GameSession(savedSession:)` provide a Mac-adapter serialization boundary without adding filesystem or Foundation dependencies to `Core/`.
- `Sources/AkalabethApp/main.swift` resumes normal saved games on launch, exposes save/resume/delete saved game menu items, and persists green/amber display treatment plus window scale settings.
- Save/resume is an intentional modern convenience; the historical notes in `README.md` state that the original distribution had no save-game option.
- `Packaging/AppIcon.svg` is the icon source, and `Scripts/package_macos_app.sh` builds `.build/release/Akalabeth.app` with bundle metadata and release executable layout.

Manual validation:

- Clean app data: remove `~/Library/Application Support/Akalabeth/SaveGame.json` and reset the app defaults domain before launch.
- Launch: `.build/release/Akalabeth.app/Contents/MacOS/Akalabeth`
- Settings persistence: change display color or window scale, quit, relaunch, and confirm the setting is restored.
- Save/relaunch: start or advance a game, use Game > Save Game, quit, relaunch, and confirm the session resumes.
- Reset/new game: use Game > New Game and confirm the old save no longer resumes.
