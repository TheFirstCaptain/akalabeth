# F-016: Audio and Feedback

## Summary

Add optional modern audio and feedback through platform adapters while keeping `Core/` portable.

## Reference

- Docs: `BOUNDARIES.md`, `docs/features/F-003.md`, `docs/features/F-010.md`

## Scope

- Map game/render/session events to optional sound effects for movement, blocked movement, combat, purchase, death, and victory.
- Add an audio setting and persist it outside `Core/`.
- Keep all audio APIs in the Mac adapter layer.

## Acceptance Criteria

- Audio can be enabled or disabled.
- Core remains free of platform audio dependencies.
- Event-to-feedback mapping is documented and covered by adapter tests where practical.

## Validation

swift build
swift test
rg -n "AppKit|SwiftUI|Metal|AVFoundation|Foundation|SDL|NS[A-Z][A-Za-z]+\\b|NSURL|Sandbox" Core
