# F-016: Audio and Feedback

## Summary

Add optional modern audio and feedback through platform adapters while keeping `Core/` portable.

## Reference

- Docs: `BOUNDARIES.md`, `docs/features/F-003.md`, `docs/features/F-010.md`

## Scope

- Map game/render/session events to optional sound effects for movement, blocked movement, combat, purchase, death, and victory.
- Add an audio setting and persist it outside `Core/`.
- Keep all audio APIs in the Mac adapter layer.

## Acceptance Criteria

- Audio can be enabled or disabled.
- Core remains free of platform audio dependencies.
- Event-to-feedback mapping is documented and covered by adapter tests where practical.

## Validation

```bash
swift build
swift test
rg -n "AppKit|SwiftUI|Metal|AVFoundation|Foundation|SDL|NS[A-Z][A-Za-z]+\\b|NSURL|Sandbox" Core
```

## Status

Implemented.

- Added `AkalabethFeedback` in the Swift session adapter to classify movement, blocked movement, combat, purchase, death, and victory outcomes after core commands.
- Added an `audioEnabled` setting persisted through `AkalabethPersistence`.
- Added Settings > Audio Feedback in the AppKit shell. Playback uses Mac-only `NSSound` system sounds and remains outside `Core/`.
- Updated Help text and README settings notes to describe optional audio feedback.
- `Tests/AkalabethMacTests/GameSessionTests.swift` covers feedback mapping and audio-setting persistence.

Feedback mapping:

| Feedback | Trigger |
| --- | --- |
| `movement` | Successful overworld movement, dungeon movement, turns, enter, or exit that changes location/orientation as expected |
| `blocked` | Rejected/invalid commands or movement commands that leave the player in the same cell |
| `combat` | Successful attack or magic command |
| `purchase` | Successful town purchase |
| `death` | Command transitions the session into death mode |
| `victory` | Command transitions the session into victory mode |

Last run: all validation checks passed.
