# F-014: Applesoft Compatibility Pass

## Summary

Audit source-shaped behavior against an Applesoft or Apple II execution reference and decide which approximations must become source-accurate.

## Reference

- BASIC lines: all behavior ranges in `ARCHITECTURE.md`
- Docs: `docs/features/F-004.md`, `docs/modernization/rnd-call-sites.md`, `docs/modernization/basic-behavior-map.md`

## Scope

- Validate RNG behavior, especially negative seeding and fractional dungeon seeds.
- Compare fixed lucky-number/level fixtures for character stats, overworld layout, dungeon layout, monster placement, combat rolls, and loot.
- Record any intentional divergence as a modernization decision.

## Acceptance Criteria

- Golden fixtures exist for representative startup, overworld, dungeon, combat, and quest states.
- Known RNG or numeric incompatibilities are either fixed or documented.
- Harness tests protect the chosen compatibility contract.

## Validation

```bash
make -C harness test
```

## Status

Implemented.

- `harness/random_tests.c` pins deterministic `RND` behavior for default generation, negative reseeding, zero-repeat semantics, integer range helpers, threshold checks, and the fractional dungeon seed expression.
- `harness/game_model_tests.c` now includes golden fixtures for lucky number `1234` and level `5`, covering startup stats, overworld tile counts and row data, dungeon layout row data, monster placement, chest loot, combat damage, and Lord British quest progression.
- `docs/modernization/rnd-call-sites.md` records the compatibility boundary: the adapter is source-shaped and deterministic, while exact Apple II numeric equivalence remains unverified without an Applesoft execution reference.

Intentional divergence:

- The portable RNG seed mixer remains a modernization decision. It preserves Akalabeth's observed `RND` call shapes and deterministic negative/fractional seed behavior, but the fixture values are the current portable contract rather than claims of bit-exact Apple II output.
