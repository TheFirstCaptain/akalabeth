# F-012: Visual Fidelity Renderer

## Summary

Replace the current simple AppKit renderer adapter with a source-shaped Apple II-style renderer for the original Akalabeth screens. This feature closes the largest visible gap left by F-009 and F-010: the app currently renders the game as a playable command stream, but not as the original UI/art presentation.

## Reference

- BASIC lines: `100-190`, `200-490`, `3087-3089`, `6000-6060`, `7000-7990`, `60080-60250`
- Screenshots: `OUTSIDE.GIF`, `TOWN.GIF`, `CHEST.GIF`, `HID-TRAP.GIF`, `LB-1.GIF`, `LB-2.GIF`, `LB-3.GIF`, `MNSTR-01.GIF` through `MNSTR-10.GIF`
- Docs: `docs/features/F-009.md`, `docs/features/F-010.md`, `docs/modernization/basic-behavior-map.md`, `BOUNDARIES.md`

## Scope

- Port overworld high-resolution tile line art from BASIC lines `100-190`.
- Port dungeon first-person perspective rendering from BASIC lines `200-490`, including depth traversal, walls, side passages, doors, ladders, trap/chest presentation, and visible monster placement.
- Port monster/chest line art from BASIC lines `300-400` and Balrog continuation lines `3087-3089`.
- Preserve the portable `Core/` boundary. AppKit-specific drawing remains in the app adapter unless the portable render-command contract must be extended with platform-neutral geometry commands.
- Keep text mode screens aligned to the original 40-column/24-row feel while still supporting modern scaling and window resizing.
- Add deterministic render fixtures for startup, town, overworld, dungeon chest, hidden trap, each monster, Lord British, death, and victory screens.

## Acceptance Criteria

- Overworld fixture rendering uses source-shaped line art for mountains, fields, towns, dungeons, and castles instead of labeled placeholder boxes.
- Dungeon fixture rendering uses depth-aware source-shaped perspective geometry rather than a fixed static tunnel sketch.
- Monster fixtures render distinct source-shaped line art for all ten monsters.
- Chest and hidden-trap fixtures visually match their historical references closely enough for manual screenshot comparison.
- Text/status positioning remains readable at all supported window scales.
- Render changes do not introduce AppKit, SwiftUI, Foundation, Metal, SDL, or other platform dependencies into `Core/`.
- Known non-pixel-perfect differences are documented before this feature is marked implemented.

## Implementation Slices

1. Overworld tile art and actor marker.
2. Dungeon perspective geometry and depth traversal.
3. Dungeon object art: walls, doors, ladders, chests, traps.
4. Monster line art for monsters `1-10`, including Balrog continuation.
5. Screenshot fixture tooling and manual comparison notes.
6. Renderer cleanup and final validation.
