# F-006: Overworld Generation, Movement, and Town Entry

## Summary

Extract overworld map generation, local 3x3 view state, movement, mountain blocking, food consumption, and location entry behavior.

## Reference

- BASIC lines: `100-190`, `1000-1587`
- Screenshots: `OUTSIDE.GIF`, `TOWN.GIF`
- Docs: `docs/modernization/legacy-inventory.md`, `ARCHITECTURE.md`

## Scope

- Model overworld tiles and local view data independently from Apple II drawing commands.
- Implement movement, blocking, dungeon/town/castle entry, and relevant status changes.
- Emit render-neutral events for visible state changes.

## Acceptance Criteria

- Harness fixtures validate movement in all four directions.
- Mountain blocking and location entry rules are covered by tests tied to source line references.
- Food or status changes caused by overworld turns are deterministic and tested.
- Rendering remains outside the portable rule implementation.

## Validation

```bash
make -C harness test
```

## Implementation Notes

Implemented. `Core/include/ak_game.h` now carries a portable `21x21` overworld tile grid matching the source `TE%(20,20)` shape, with named tile values for open space, mountains, fields, town, dungeon, and castle.

`Core/src/ak_game.c` implements source-backed overworld generation from BASIC lines `30-50`, town exit into the overworld, four-direction overworld movement from lines `1100-1410`, mountain blocking, overworld food consumption/starvation from line `1090`, and `ENTER` handling for town/dungeon/castle tiles from lines `1500-1515`.

Dungeon entry initializes the portable dungeon mode, coordinates, level, and facing, then returns `AK_GAME_RESULT_UNSUPPORTED_RULE` because dungeon generation/navigation remain F-007. Castle entry transitions to quest/castle state and returns `AK_GAME_RESULT_UNSUPPORTED_RULE` because Lord British quest rules remain F-008.
