# F-002: BASIC Behavior Map and Extraction Fixtures

## Summary

Create a source-backed behavior map that ties major gameplay areas to exact Applesoft BASIC line ranges, representative fixtures, screenshots, and known uncertainties before extracting more game rules.

## Reference

- BASIC lines: `0-99`, `100-190`, `200-590`, `1000-1700`, `2000-2090`, `4000-4999`, `6000-6060`, `7000-7990`, `60000-60250`
- Screenshots: `STARTUP1.GIF`, `STARTUP2.GIF`, `STARTUP3.GIF`, `OUTSIDE.GIF`, `TOWN.GIF`, `LB-1.GIF`, `LB-2.GIF`, `LB-3.GIF`, `CHEST.GIF`, `HID-TRAP.GIF`, `MNSTR-01.GIF` through `MNSTR-10.GIF`
- Docs: `docs/modernization/legacy-inventory.md`, `ARCHITECTURE.md`, `ENGINEERING_HARNESS.md`
- Behavior map: `docs/modernization/basic-behavior-map.md`

## Scope

- Add structured documentation or harness fixtures that identify inputs, state variables, outputs, and Apple II dependencies per behavior area.
- Extend the source listing harness where useful to verify expected line references and data tables.
- Record source ambiguities as explicit open questions instead of resolving them by guesswork.

## Acceptance Criteria

- Every major line range from `ARCHITECTURE.md` has a corresponding behavior-map entry.
- Each entry identifies whether it is ready for extraction, blocked by Apple II platform behavior, or needs additional characterization.
- The harness verifies the presence of high-value source anchors used by the behavior map.
- No historical source files or GIF references are modified.

## Validation

```bash
make -C harness test
```
