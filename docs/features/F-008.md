# F-008: Combat, Magic, Death, and Quest Progression

## Summary

Extract attack rules, weapon handling, magic amulet effects, monster damage, death handling, Lord British quests, and win condition into portable tested code.

## Scope

- Implement combat command resolution and damage/status events.
- Implement magic amulet outcomes and any class or inventory restrictions found in the listing.
- Implement death state, quest assignment, quest completion, and victory transitions.

## Acceptance Criteria

- Harness tests cover hit, miss, kill, damage, magic, death, quest assignment, quest completion, and final victory paths.
- Quest target names and monster ordering match the source data tables.
- All random combat outcomes can be driven by deterministic fixtures.
- Intentional modernization differences, if any, are recorded in the feature file before implementation is marked complete.

## Implementation Status

Implemented in the portable core.

- `AK_GAME_COMMAND_ATTACK` resolves source-shaped weapon hit, miss, damage, kill reward, and quest credit rules.
- `AK_GAME_COMMAND_CAST_MAGIC` resolves amulet ladder, magic attack, bad outcome, and charge-loss rules.
- Dungeon monster turns include attack damage plus thief item theft and gremlin food theft.
- Castle entry enters quest mode, acknowledgement assigns or advances Lord British quests, and the Balrog quest completion enters `AK_GAME_MODE_VICTORY`.
- Monster names are exposed through `ak_game_monster_name` in source table order.

Intentional modeling notes:

- Player name entry and Lord British prose remain renderer/input-adapter concerns until F-009 defines the render command contract.
- The BASIC axe throw-versus-swing prompt is not represented as a separate command field yet; the current core command models weapon choice and uses bow/magic for ranged targeting.
