# F-013: Complete Interaction Prompts

## Summary

Complete the original multi-step input and prose flows that were simplified during the first playable port.

## Reference

- BASIC lines: `1650-1697`, `6000-6060`, `7000-7990`, `60200-60250`
- Screenshots: `LB-1.GIF`, `LB-2.GIF`, `LB-3.GIF`, `TOWN.GIF`
- Docs: `docs/features/F-008.md`, `Sources/AkalabethMac/GameSession.swift`

## Scope

- Add explicit prompt states for attack weapon choice, axe throw-versus-swing, magic amulet choice, player name entry, Lord British adventure consent, and acknowledgement prompts.
- Preserve original prose flow where the BASIC listing presents it.
- Keep modern direct-key shortcuts only when documented as adapter conveniences.
- Extend session tests to cover multi-key flows.

## Acceptance Criteria

- Combat prompts can be driven step-by-step like the original.
- Lord British name/adventure flow preserves the source sequence.
- Invalid prompt input produces visible feedback and does not corrupt state.
- Swift tests cover all prompt states and cancellation/reset behavior.

## Implementation Notes

Implemented in the Mac session adapter.

- `GameSession` now exposes `AkalabethPrompt` for active prompt state.
- Dungeon attack starts with the original `WHICH WEAPON` prompt when pressing `A`.
- Axe attacks ask for `TO THROW OR SWING` before resolving.
- Mage amulet use asks for the original `1-LADDER-UP`, `2-LADDER-DN`, `3-KILL`, `4-BAD??` choice.
- First Lord British quest entry asks for a player name, then asks for adventure consent before sending the quest acknowledgement to the portable core.
- Tests cover attack weapon prompting, axe style prompting, mage magic choice, and first quest name/consent flow.

Known residual differences:

- Player name is currently adapter prompt input only; `Core/` does not yet store or render the name in Lord British prose. That can be addressed with a future core text/prose feature if full name persistence is required.
- Thrown axe handling uses the existing core ranged-attack path after decrementing an axe because the current core command model does not carry an explicit throw-versus-swing field. A future compatibility pass can promote this into `Core/`.
- Existing direct-key conveniences for some commands remain available where they predated F-013.
