# F-018: Packaging and Release Readiness

## Summary

Prepare the modern Akalabeth app for credible local distribution and operator-owned signing/notarization.

## Reference

- Docs: `docs/features/F-011.md`
- Code: `Scripts/package_macos_app.sh`, `Packaging/AppIcon.svg`

## Scope

- Harden release packaging and validation scripts.
- Add release checklist for clean install, settings, save/resume, reset, and fixture smoke tests.
- Review bundle metadata, icon, About text, and app naming.
- Document signing and notarization commands without requiring credentials in the repository.

## Acceptance Criteria

- Release package can be built from a clean checkout.
- Packaged app passes smoke launch and manual validation checklist.
- Signing/notarization steps are documented for the release operator.

## Validation

```bash
swift build -c release
swift test
make -C harness test
Scripts/package_macos_app.sh
.build/release/Akalabeth.app/Contents/MacOS/Akalabeth --smoke-test
```

## Status

Implemented.

## Implementation Notes

- `Scripts/package_macos_app.sh` now accepts `--require-clean` and `--skip-smoke-test`, supports release metadata through `AKALABETH_BUNDLE_ID`, `AKALABETH_VERSION`, and `AKALABETH_BUILD_NUMBER`, and supports optional operator-owned signing through `AKALABETH_CODESIGN_IDENTITY`.
- The package script builds the release executable, assembles `.build/release/Akalabeth.app`, lints `Info.plist`, records the executable file type, verifies arm64 output, runs the packaged `--smoke-test`, and verifies signatures when a signing identity is supplied.
- `docs/RELEASE.md` documents clean checkout validation, clean-install manual checks, settings persistence, save/resume/reset validation, fixture smoke checks, signing, notarization, stapling, and release note capture.
- `README.md` links the release checklist from the local packaging instructions.
