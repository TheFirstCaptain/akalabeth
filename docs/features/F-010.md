# F-010: Native macOS Apple Silicon App Shell

## Summary

Create a native macOS app shell that runs on Apple silicon, hosts the portable core, translates keyboard/menu/window events into core commands, and renders Apple II-style output.

## Reference

- BASIC lines: adapters around all extracted core behavior
- Screenshots: all startup, overworld, town, Lord British, chest, trap, and monster GIF references
- Docs: `ARCHITECTURE.md`, `BOUNDARIES.md`, `DECISION_LOG.md`, `docs/features/F-003.md`, `docs/features/F-009.md`

## Scope

- Add a modern macOS project or Swift Package target for an arm64-native app.
- Implement app lifecycle, window setup, keyboard input mapping, menu commands, and a renderer adapter.
- Link to the portable C core without introducing macOS framework dependencies into `Core/`.
- Include a debug route or development command for launching deterministic fixtures.

## Acceptance Criteria

- The app builds and runs natively on Apple silicon as `arm64`.
- The first playable flow supports startup, town/store, overworld, dungeon navigation, combat, and quest screens implemented by previous core features.
- Keyboard input maps to documented game commands and menu items cover expected Mac app actions such as quit, new game, and reset.
- Manual validation records launch command, tested flows, and known gaps.

## Implementation Status

Implemented as a Swift Package.

- `Package.swift` defines a `CAkalabeth` C target for the portable core, an `AkalabethMac` Swift library for command/session orchestration, and an `AkalabethApp` AppKit executable.
- `Sources/AkalabethApp/main.swift` creates the native menu, window, keyboard handling, fixture menu items, and an AppKit renderer for `ak_render_game` command buffers.
- `Sources/AkalabethMac/GameSession.swift` maps startup input, movement, town purchases, combat keys, magic choices, reset, and deterministic fixtures to the portable C core.
- `Tests/AkalabethMacTests/GameSessionTests.swift` covers startup-to-town flow, representative fixture launches, town input, dungeon command mapping, and render-buffer availability.

Manual validation:

- Build: `swift build`
- Smoke launch without leaving a window open: `.build/arm64-apple-macosx/debug/AkalabethApp --smoke-test`
- Interactive launch: `.build/arm64-apple-macosx/debug/AkalabethApp`
- Deterministic fixture launch: `.build/arm64-apple-macosx/debug/AkalabethApp --fixture=dungeon`
- Menus cover quit, new game, reset, and debug fixtures for town, overworld, dungeon, and quest.

Known gaps:

- The SwiftPM target currently produces an AppKit executable, not a signed `.app` bundle; packaging is deferred to F-011.
- Rendering is intentionally a simple adapter over the F-009 command stream, not pixel-perfect Apple II output.
