# F-007: Dungeon Generation, Navigation, and Encounters

## Summary

Extract dungeon generation, player orientation, forward/turn navigation, traps, ladders, chests, monster placement, and monster turn behavior.

## Reference

- BASIC lines: `200-590`, `1000-1587`, `2000-2090`, `4000-4999`
- Screenshots: `CHEST.GIF`, `HID-TRAP.GIF`, `MNSTR-01.GIF` through `MNSTR-10.GIF`
- Docs: `docs/modernization/legacy-inventory.md`, `docs/modernization/validation-strategy.md`

## Scope

- Model dungeon grid data, depth, orientation, encounter contents, and turn sequencing.
- Extract trap, ladder, chest, and monster placement state transitions.
- Add deterministic fixtures for dungeon generation and monster turn outcomes.

## Acceptance Criteria

- Fixed seeds generate reproducible dungeon fixtures.
- Navigation tests cover turning, forward movement, blocked movement, ladder transitions, and dungeon exit behavior.
- Encounter tests cover chest/trap discovery and at least one monster placement path.
- Monster movement and attack events are represented without requiring rendering.
