# F-005: Startup, Character Creation, and Store Rules

## Summary

Extract the startup prompts, character qualities, class selection, inventory initialization, and town store purchase rules into portable tested code.

## Reference

- BASIC lines: `60000-60250`, `0-99`
- Screenshots: `STARTUP1.GIF`, `STARTUP2.GIF`, `STARTUP3.GIF`, `TOWN.GIF`
- Docs: `docs/modernization/legacy-inventory.md`

## Scope

- Implement prompt flow state for lucky number, level, generated qualities, and class selection.
- Extract item names, prices, inventory fields, status display values, and purchase validation rules.
- Preserve original behavior unless a deviation is documented as an intentional modern change.

## Acceptance Criteria

- Fixed seeds produce repeatable character-generation fixtures.
- Store commands update gold, inventory, and status exactly according to the extracted rules.
- Invalid input and unaffordable purchase behavior are represented in command results.
- Harness tests cover at least one complete startup-to-town flow.

## Validation

```bash
make -C harness test
```
